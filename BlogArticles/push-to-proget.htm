Push to Proget

A Universal Package Manager is a key link in a DevOps Toolchain, and often the central hub for an organization's software development.

Because it's so vital, it is important to ensure that your UPM is maintained and updated with the latest packages and build artifacts as they are created, adapted, and developed. The Push to ProGet Visual Studio Extension allows you to maintain and update a Visual Studio Project feed right from the IDE.

ProGet is a Universal Package Manager that allows you to host both in-house and third party packages, libraries, and applications locally. Follow along with the Push to ProGet Visual Studio Extension tutorial, and learn how to push directly from Visual Studio, to a designated Universal ProGet feed.

Set up is easy, simply download the Visual Studio Extension in the normal manner, and from the solution explorer, right click on the project, and select Push to ProGet.

Then, fill out the information needed in the few dialog boxes.

Lastly, confirm and push your package, and Visual Studio will build the chosen project, package everything together as a Universal Feed Package, and push it to ProGet.

Thats it! Once packages are in ProGet, it's easy to import those packages to BuildMaster to be deployed as part of a Release Package, or pulled to Otter to provision infrastructure.
