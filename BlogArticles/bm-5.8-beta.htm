Introducing Plan Versioning, Text Templating, and an updated LDAP and active directory integration, in beta now!

Deployment plans are at the heart of BuildMaster's automation, and allow users to direct and structure their releases either visually, from a drag-and-drop editor, or from a text-based editor. Though this makes creating deployment plans easier, it can be challenging for larger development teams to know when a plans' configuration has been changed.

With Plan Versioning, a history of edits is maintained within BuildMaster; allowing approved team members to view the version history, or even preform a rollback to a previous version. You can also simply view the differences between two versions from the Plan Version Listing page. The differences between the two selected plans will be highlighted for quick identification of the changes that have been made. Using plan versioning will bring your team confidence when making changes, and allow for clarity throughout your deployment process.

Our new and improved LDAP and Active Directory integration, makes maintaining user and group permissions easier than before. Complex, multi-domain active directory structures are now easily catered to, as our new integration allows permissions to be defined in a third-party LDAP directory, or an active directory domain forest. If you choose to use an active directory domain, you can also enable Windows Integrated Authentication, removing the web-based login prompt.

Also coming in BuildMaster 5.8, we've simplified the deployment of configuration files. Traditional configuration file assets did not allow conditionals or loops, but since text templates are transformed using a simplified OtterScript syntax, they allow the use of if/else conditionals, loops blocks, and access to all variable functions. Simply use Text Templating combined with Configuratation Variables as a deployment mechanism. Though there is still a time and place for configuration file assets, text templating provides a second option for a powerful yet simple replacement.
