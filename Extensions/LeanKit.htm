Using LeanKit Kanban with BuildMaster

LeanKit Kanban is a hosted service, so your BuildMaster server must have access to the Internet for this extension to work. The LeanKit Kanban extension includes a BuildMaster Issue Tracking Provider that allows you to treat a Kanban board as a BuildMaster application's issue tracker. If you are unfamiliar with configuring Issue Tracking Providers in BuildMaster, see Integrating an Issue Tracker with BuildMaster.

Configuring the Kanban Provider

The Kanban Provider in BuildMaster requires a few pieces of information to connect to the hosted service.

Issues in BuildMaster are always associated with a Release, so you must tag your cards with an appropriate Release Number for BuildMaster to match them up with active Releases. The default format of rel-%RELNO% will allow you to use tags like rel-1.0 or rel-1.2.5 (depending on your BuildMaster Release Numbering Scheme), but you may use any format you would like.

The name of your LeanKit account is also required, along with logon credentials which BuildMaster can use to view and/or update cards.

Associating a Board with an Application

Select Issue Tracker from the Settings drop-down menu on the application navigation bar that you would like to associate with your Kanban board, and choose the newly-created LeanKit Kanban Provider from the dropdown box. Next, select the board from the Board dropdown box and click Save.

Issue Status and Actions

Once associated, you should now be able to view cards as issues on the Release Overview page of your application. All cards on the selected board with a tag matching the BuildMaster Release Number will be displayed in the Issues section along with a link to the specific card in LeanKit Kanban.

Card titles are displayed in bold while the lane which the card currently occupies is displayed next to the title (in parentheses). If the card has a description, it is displayed below the title. Cards occupying the final lane of a board are regarded as closed issues in BuildMaster – they will be displayed with a green check mark instead of the standard blue diamond icon.

The LeanKit Kanban Provider includes support for all of the standard Issue–Tracking BuildMaster actions:

  • Append Issue Description – Appends text to the description of a card
  • Change Issue Status – Moves all of the cards from one lane to another lane
  • Close Issues – Moves all of the cards to the final lane

Additionally, the Issue Tracker Status Promotion Requirement may be used to block promotion of a Release until all of the cards are in a specific lane.
