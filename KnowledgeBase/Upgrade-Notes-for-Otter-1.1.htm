Otter 1.1 is minor release with the following changes:

  • Support for SSH-based Servers
  • Git-based Rafts
  • Various bug fixes
  • New SDK
  • New API

Upgrade Impact

Because the changes are additive, the upgrade risks are negligible: there is a minor database schema change, and you can mitigate this risk by backing-up your Otter instance prior to upgrade.

Upgrade Process

Otter may be upgraded by simply running the latest installer. The agents will automatically update when the service starts again.

Rollback

Because there are database changes, a rollback will require uninstalling Otter, and then restoring your Otter instance.
