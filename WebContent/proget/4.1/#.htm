ProGet is a Universal Package Manager that lets you host and manage not only your in-house packages, but also an ever growing list of external repositories (now including PowerShell!). It was designed from the ground-up to be fast and easy to use, for both personal and enterprise use.

Upgrading? Make sure to check out Upgrade Notes for ProGet 4.1

Introducing Multi-site Replication

ProGet Enterprise users now have the ability to mirror feeds across multiple ProGet servers, both locally and globally. Implementing the multi-site replication requires specific feed configurations; one feed must be configured as allow other feeds to replicate with and one or more feeds configured to replicate from external feed.

This allows for a lot of different configurations, including synchronizing multiple ProGet servers across the globe.

In this example, Los Angeles is configured to allow other feed to replicate with it, while Cleveland, and Tokyo are configured to replicate from external feeds.

Want more info? For a more in-depth explanation of ProGet's multi-site replication functionality, checkout the documentation.

PowerShell = Supported!

After listening to user feedback, we've officially added PowerShell as a supported feed type. Now you can seamlessly host all your PowerShell scripts and modules in one place. Plus, keep an eye out for this icon in your feeds repository!
