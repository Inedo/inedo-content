Asset Directories

For many organizations ProGet acts as a central-hub for development and applications packages. With the addition of Asset Directories, users are now able to utilize all the security features in ProGet for not only packages, but arbitrary file storage.

Utilizing Asset Directories

ProGet Asset Directories gives you the versatility of a file share, but with a built-in web server and a powerful REST API. As part of ProGet, Asset Directories can be easily managed along with all of your important packages.

Asset Directories can be used to:

  • Host simple internal static websites
  • Share pictures with work colleagues
  • Store a self-contained installer package

Creating, Managing, and Adding

When creating a new Asset Directory, the only required field is its name. Like Feeds, Asset Directories must be uniquely named, and should be easily represented in a URL (in other words, avoid characters like /, : , or spaces). As soon as you have named the new Directory, you can start uploading files to it.

Files can be added to an Asset Directory in a number of ways. You can:

  • Upload a single file using a browser
  • Upload a .zip or a .tar.gz file using your browser and expand its contents into the asset directory
  • Use the Asset Directory API to upload on or more files to the asset directory
  • Upload a large file or multiple files at once
