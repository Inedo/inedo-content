全てのパッケージを一括して管理

アプリケーションとコンポーネント

ユニバーサルパッケージ を使用して、アプリケーションとコンポーネント、およびコンテナ化されたソフトウェアのDockerイメージを配布します。

ディベロッパー ライブラリ

C＃/ .NET(NuGet)、JavaScript / NodeJS(npm)、Ruby(Gems)、Java(Maven)のいずれを使用していても、開発チーム間で事前にビルトされたテスト済みのコードを共有できます。

システム ソフトウェアと構成

サードパーティのWindowsソフトウェア(Chocolatey)、PowerShellパッケージ、およびPowerShell DSCリソースを管理します。

外部パッケージと内部パッケージを管理し整理

パッケージを整理して、他のグループや顧客に配布するためのフィードを作成します。

複数のフィード（内部と外部の両方）をコネクタと組み合わせることで、フィードの利用を一つにし、外部フィードが利用できない場合でも外部パッケージにアクセスできます。

オープンソースパッケージのスキャンと検証

ProGetが準拠していないパッケージをブロックするために使用するライセンスポリシーを定義することで、コストのかかるオープンソースライセンス違反を避けることができます。

パッケージの脆弱性をスキャンし、設定したルールに基づいて危険または安全でないと判断されたものへのアクセスを遮断します。

プロダクションなどの適切な環境で、承認され、検証されたパッケージのみが使用されるように、フィード間でパッケージをプロモートします。

パッケージの展開と使用履歴

テスト、導入、および必要に応じてロールバックを自動化するHedgehogの再利用可能なワークフローを統合することにより、アプリケーションおよびコンポーネントパッケージをサーバーとクラウドにデプロイします。

ProGetは、BuildMaster、Otter、Octopus Deploy、独自のカスタムスクリプトなどの他のツールからもデプロイを記録するので、どのパッケージがどのサーバーにデプロイされているかが一目瞭然です。

パッケージの利用統計を用いて、どのユーザーとサーバーがどのパッケージにアクセスしているかを確認し、開発ライブラリー、スクリプト、または他のタイプのパッケージのアップデートや停止に関する情報に基づいた決定をおこなうことが可能です。

パッケージが追加、プロモート、デプロイ、および削除されたときに自動デプロイ、変更、アクティビティの監査を行うようにwebHookを設定します。

サーバーやクラウドに信頼できるインフラストラクチャをビルド

必要な負荷の分散を実施し、世界中のさまざまなチームやサーバーを構築するために必要な数のWebサーバーノードを追加します。

Amazon S3またはAzure Blobストレージを活用して、パッケージファイルストレージの安価で最適なスケーリングを実現できます。

世界中のProGetインスタンス間でフィードをレプリケケーションすることで、複雑なグローバルインフラストラクチャを構築することなく、異なるチームがフィードを共有できるようにします。

ProGetのハイ アビリティ モードが障害の全くないことを確認し、ハードウェア上にエラーがある場合にもパッケージがアクセスすることを可能にします。

全ての Inedo のプロダクトは、共通の信頼性の高いテクノロジーを用いて設計されています。

Inedo Execution Engine

Inedo Execution Engine は、インフラストラクチャの自動化とオーケストレーション専用に開発されたもので、何百ものサーバーに展開して実行することを容易にします。また、問題個所を速やかに把握するために詳細なログを、（詳細なデバッグ情報を特定のユーザーから隠しながら）提供します。

Inedo Agent

軽量で高度に最適化された、サーバーに接続可能なエージェントを、Inedo プロダクトによって安全に編成できます。 容易にデプロイとアップグレードができ、変更のリスクを最小限に抑えるために、後方互換性と前方互換性を備えています。

Inedo SDK

Inedo プロダクトの機能の多くは拡張機能を通して提供され、拡張機能を作成することで、機能を追加や変更することができます。 これはシェア SDK であるため、コンポーネントを一度作成することで、全ての Inedo プロダクトに同じ機能を導入することが可能です。
