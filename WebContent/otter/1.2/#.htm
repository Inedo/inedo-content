1.2 has arrived!

Note: This is not the latest version of Otter.

Otter maintains your servers' configuration using an infrastructure as code approach, and with Otter 1.2, you now have the ability to import and export Otter's very own configuration as code. This makes sharing and maintaining your infrastructure configuration that much easier.

If you're upgrading, make sure to check out Upgrade Notes for Otter 1.2.

Import and Export Configuration

You can export all of your instance's infrastructure configuration (servers, environments, and server roles) as a JSON file.

You can also import a configuration file as well; this allows you to quickly configure or make massive changes to Otter's infrastructure configuration. And with the "dry run" option, you can see the changes that would be made before actively making the changes.

Configuration Synchronization

It's quite common to Otter for managing server configuration, and BuildMaster for application release automation; because they both use the same set of servers, you can synchronize the infrastructure between multiple instances.

When you enable infrastructure synchronization, your Otter instance connects to either a BuildMaster or Otter instance, pulls configuration from that instance, and updates the infrastructure configuration as needed.

Infrastructure Management API & API Keys

The Infrastructure Management API Endpoints offer a simple mechanism for querying, creating, and updating server, environment, and server role entries; they are intended to automate the set-up and management of an Otter instance.

In addition, you can now create specific API Keys which can control which functionality a may be used by which integration is given the key.
