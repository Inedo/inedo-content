BuildMaster is an Application Release Automation tool built for DevOps. You can create complex release pipelines all managed from one central dashboard.

If you're upgrading, make sure to check out Upgrade Notes for BuildMaster 5.5.

Otter Integration

To help strengthen the link between your applications and infrastructure configuration, we've included three new integration points into Otter.

  • Set Variable Value Operation - Creates or assigns a configuration variable in Otter, such as a release number of a required package, a last deployed date, or a required application
  • Remediate Drift Operation - Checks configuration status and if drifted, triggers a remediation job in Otter.
  • Drift Status Gate - Verifies that a specified Otter server or role is in a specified drift status prior to deployment.

Pipeline Variables and Bulk-edits

You can now specify variables in a pipeline and a pipeline stage. This allows for improved reusability of deployment plans.

Additionally, you can bulk-edit sets of configuration as JSON.

The Release & Package Deployment API

This new API endpoint offers a simple mechanism for creating releases, creating release packages, and deploying release packages.

See API documentation to learn more.
